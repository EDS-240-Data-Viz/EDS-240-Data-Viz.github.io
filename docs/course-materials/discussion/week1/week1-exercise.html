  1. Clean / wrangle the data! Using the packages loaded in step 1, perform the following operations (NOTE: Each step below corresponds to a single line of code):
    1. convert all column names from CamelCase to snake_case (for ease of readability and to adhere to the Tidyverse style guide recommendation)
    2. remove "AM" from the job_start_date column observations
    3. convert observations in the job_start_date column from character strings to datetime objects and save them to a new column named datetime_start
    4. add a column named year that contains only the year from datetime_start
    5. keep only the necessary columns (datetime_start, year, state_name, well_name, total_base_water_volume)
    6. convert all observations in the state_name column to Title Case
    7. remove any rows where the state name ends in ?
    8. convert any state abbreviations to full names in the state_name column (Hint: use a combination of the mutate(), ifelse() & usdata::abbr2state() functions)
    9. correct any misspelled state names in the state_name column (Note: figure out how to update the names of a few misspelled states, the feel free to copy the complete code for this step from the expandable section, below)
    10. remove any rows that do not have a true US state name in the state_name column
    11. add a column named state_abb with just US state abbreviations
    12. move the state_abb column so that it sits immediately after the state_name column
    13. rename total_base_water_volume as total_base_water_volume_gal for clarity
    14. remove any observations that donâ€™t have a measurement for total_base_water_volume_gal (i.e. NA)
    15. reorder rows from earliest to most recent datetime_start values
